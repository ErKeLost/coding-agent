Reads a file from the sandbox filesystem. If the User provides a path to a file assume that path is valid. It is okay to read a file that does not exist; an error will be returned.

Usage:
- Paths are resolved under the current workspace root. Relative paths resolve from workspace root; legacy `/workspace/...` paths are also accepted.
- This tool reads the file contents directly. It does not add synthetic line numbers or paging hints.
- Use this when you already know the file path and want the file contents.
- If you need to locate files first, use `list`, `glob`, or a shell search command.
- You have the capability to call multiple tools in a single response. It is always better to speculatively read multiple files as a batch that are potentially useful.
- If you read a file that exists but has empty contents you will receive a system reminder warning in place of file contents.
- You can read image files using this tool.
