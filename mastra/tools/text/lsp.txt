Interact with Language Server Protocol (LSP) servers to get code intelligence features.

Supported operations:
- documentSymbol: Get all symbols (functions, classes, variables) in a document
- workspaceSymbol: Search for symbols across the entire workspace
- sandboxSymbols: Search for symbols across the entire sandbox
- completions: Get completion items at a position
- didOpen: Notify the LSP server that a file was opened
- didClose: Notify the LSP server that a file was closed
- start: Start the LSP server
- stop: Stop the LSP server

All operations require:
- sandboxId: The sandbox to run the LSP operation in

Some operations additionally require:
- filePath: Required for file-based operations (documentSymbol, didOpen, didClose, completions)
- line/character: Required for position-based operations (completions)
- query: Optional for sandboxSymbols/workspaceSymbol (defaults to empty)

Optional parameters:
- languageId: typescript | javascript | python (default: typescript)
- projectPath: LSP project root (default: /workspace)

Notes:
- line/character are 1-based in this tool and converted to 0-based for the LSP server.
- workspaceSymbol is an alias for sandboxSymbols.

Note: LSP servers must be configured for the file type. If no server is available, an error will be returned.
