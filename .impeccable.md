## Design Context

### Users
General developers using a local-first AI coding workspace for debugging, refactoring, planning, file inspection, and agent steering. They use the product during real work sessions, often for long periods, and expect the interface to stay calm, readable, and operationally precise.

### Brand Personality
Fresh, focused, and capable.

The interface should feel confident and clean rather than flashy. It should carry some of the crispness and ease of Raycast, but translated into a desktop coding workspace with stronger information density and clearer execution-state feedback.

### Aesthetic Direction
Dual-theme unified language.

The product should support both light and dark themes with the same visual grammar:
- fresh, lightly editorial surfaces in light mode
- restrained operator-console depth in dark mode
- compact spacing, sharp hierarchy, and low-noise chrome in both

Reference direction:
- Raycast-like clarity, compactness, and tactile controls
- local coding workspace, not a generic SaaS dashboard

Anti-direction:
- playful consumer chat styling
- glass-heavy or glow-heavy AI aesthetics
- purple/cyan neon dark themes
- card-on-card dashboard clutter

### Design Principles
1. Prioritize operational clarity over decoration.
2. Keep chat, tools, plans, and workspace controls in one visual language.
3. Use accent color surgically for state and intent, never as wallpaper.
4. Make dual themes feel like siblings, not two unrelated products.
5. Preserve calm during long sessions with compact, consistent, low-noise surfaces.
